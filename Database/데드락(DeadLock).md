# 데드락 해결 방법

데드락이란 두 개 이상의 트랜잭션이 각각의 작업을 완료하기 위해 다른 트랜잭션이 점유하고 있는 자원을 대기하는 상태를 말합니다. 이로 인해 두 트랜잭션이 서로 무한히 대기하는 상황이 발생하게 됩니다. 

데드락을 해결하기 위한 주요 방법들은 다음과 같습니다:

1. **데드락 예방**: 데드락의 발생 조건을 제거하여 데드락이 발생하지 않도록 하는 방법입니다.
    - **상호 배제**: 하나의 자원은 한 번에 하나의 트랜잭션만 사용할 수 있습니다.
    - **점유와 대기**: 자원을 이미 점유하고 있는 트랜잭션이 다른 자원을 요청할 때, 점유한 자원을 모두 해제하고 다시 모든 자원을 요청하는 방식으로 데드락을 예방합니다.
    - **비선점**: 트랜잭션이 이미 점유하고 있는 자원은 다른 트랜잭션에게 선점되지 않습니다.
    - **순환 대기**: 자원을 번호 순서대로 할당하거나, 특정 순서를 지정하여 요청하는 방식으로 순환 대기를 방지합니다.

2. **데드락 회피**: 데드락이 발생할 가능성을 확인하고 회피하는 방법입니다.
    - **자원 할당 그래프**: 데드락 발생 가능성을 판별하기 위해 자원 할당 그래프를 사용하여 순환 구조를 탐지합니다.
    - **은행원 알고리즘**: 자원 요청 시 시스템의 상태를 체크하여 데드락의 가능성이 있을 때 해당 요청을 거절하는 알고리즘입니다.

3. **데드락 탐지 및 복구**: 데드락이 발생했는지 주기적으로 확인하고, 발견 시 복구하는 방법입니다.
    - **데드락 탐지 알고리즘**: 시스템에서 데드락이 발생했는지 주기적으로 확인합니다.
    - **데드락 복구**: 데드락이 탐지되면 해당 트랜잭션을 중단하거나, 트랜잭션에 할당된 자원을 회수하여 데드락을 해결합니다.

4. **시간 제한**: 트랜잭션에 시간 제한을 설정하여 일정 시간 동안 자원 할당을 받지 못하면 자동으로 실패로 처리합니다. 이로 인해 데드락이 일정 시간 동안만 지속되도록 합니다.

5. **자원 요청 순서 변경**: 트랜잭션들이 자원을 요청하는 순서를 변경하여 데드락을 회피하는 방법입니다.
